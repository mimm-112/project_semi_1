# 산업현장 근로자 피로·각성 상태 판별 서비스
## 세미나 1 발표 자료 (상세본)

---

**프로젝트명:** PVT 기반 근로자 피로·각성 상태 판별 및 숨은 위험 탐지 서비스
**데이터:** OpenNeuro ds000201 (주 데이터) + DROZY (검증 데이터)
**핵심 기여:** 주관·객관 피로 지표의 독립성을 규명하고, 이를 기반으로
"본인이 인지하지 못하는 각성 저하(숨은 위험)"를 식별하는 2×2 분류 체계 제안

---

# 1. 문제 제기 (Problem Definition)

## 1.1 사회적 배경

산업현장에서 근로자의 피로는 단순한 컨디션 문제가 아니라 **안전과 직결되는 위험 요인**이다.
피로가 누적되면 다음과 같은 변화가 나타난다.

- **반응 속도 저하:** 위험 상황에 대한 대응이 느려진다
- **주의력 붕괴(lapse):** 순간적으로 주의를 놓치는 현상이 빈번해진다
- **상황 인식 저하:** 주변 위험 신호를 놓치기 쉬워진다

이러한 변화는 기계 오조작, 낙상, 협착, 충돌 등 중대 산업재해의 직접적 원인이 된다.
특히 교대근무·야간작업이 많은 제조·건설·운송 현장에서 피로로 인한 사고 위험은
상시적으로 존재한다.

## 1.2 핵심 문제: "인지되지 않는 위험"

기존의 피로 관리가 놓치고 있는 가장 중요한 지점은 다음과 같다.

> **근로자 본인이 자신의 각성 저하를 인지하지 못하는 경우가 존재한다.**

사람은 자신의 피로 상태를 항상 정확히 감지하지 못한다. "아직 괜찮다"고 느끼지만
객관적으로는 이미 반응 속도가 떨어지고 주의력이 저하된 상태일 수 있다.
이 경우:

- 근로자는 스스로 위험하다고 느끼지 않으므로 **휴식을 취하지 않는다**
- 안전관리자는 근로자의 실제 상태를 파악할 객관적 수단이 없어 **개입하지 못한다**

즉, 가장 위험한 근로자가 오히려 관리의 사각지대에 놓인다. 본 프로젝트는
이 "인지되지 않는 위험"을 데이터로 식별하는 것을 핵심 과제로 삼는다.

## 1.3 기존 접근의 한계

| 기존 방식 | 한계 |
|---|---|
| 주관적 자기보고 (설문) | 본인이 인지 못하면 무의미. 과소·과대 보고 |
| 고가 생체신호 장비 (EEG 등) | 비용·착용 부담으로 일상 현장 적용 어려움 |
| 집단 일괄 기준 | 개인차(원래 반응이 느린 사람)를 반영 못 함 |

## 1.4 해결 방안 (Resolution)

본 프로젝트는 세 가지 축을 결합하여 문제를 해결한다.

1. **객관 지표(PVT):** 심리운동 각성 검사로 반응속도·주의력을 정량 측정
   - PVT는 각성도 측정의 국제 표준(gold standard)이며 3분 이내 간편 측정 가능
2. **주관 지표(설문):** 본인이 느끼는 피로·컨디션
3. **개인특성:** 불안·스트레스·수면습관 등 피로 취약성 관련 배경 변수

핵심은 **주관과 객관을 하나로 합치지 않고, 두 축의 어긋남(불일치)을 포착**하는 것이다.

## 1.5 솔루션 (Solution)

- PVT 3분 측정 + 간편 설문으로 근로자의 피로 상태를 4가지로 자동 분류
- 특히 **"객관적으로는 위험하나 본인은 인지하지 못하는" 근로자(숨은 위험)**를 식별
- 안전관리자 대시보드를 통해 팀 전체의 위험 상태를 시각화하고 우선순위 경고

# 2. 솔루션 컨셉 (Solution Strategy)

## 2.1 핵심 통찰: 피로는 단일 지표가 아니다

본 프로젝트의 출발점이자 가장 중요한 발견은 다음과 같다.

> **피로는 하나의 숫자로 표현되지 않으며, 서로 독립적인 두 개의 축으로 구성된다.**

- **객관 축 (수행 능력):** PVT가 측정하는 각성·주의력. 사고 위험과 직결.
- **주관 축 (자각 상태):** 본인이 느끼는 피로·신체 컨디션.

우리는 데이터 분석을 통해 이 두 축이 **통계적으로 독립**임을 확인했다
(ds000201에서 PVT와 주관 지표 SicknessQ의 상관 r≈0, 카이제곱 p=0.143).
이는 "본인이 피곤하다고 느끼는 것"과 "실제 각성이 저하된 것"이
서로 다른 정보임을 의미한다.

### 왜 두 축이 독립인가 — DROZY 검증

두 축의 독립성이 "측정 오류"가 아니라 "실제 현상"임을 별도 데이터로 검증했다.

- DROZY 데이터에서 **졸림 척도(KSS) ↔ PVT = r=0.63 (강한 상관, p<0.001)**
- 즉, 주관 지표라도 "졸림"을 재면 객관 각성과 잘 연관된다
- ds000201의 SicknessQ가 PVT와 무관한 것은, SicknessQ가 "졸림"이 아니라
  "신체적 불편감(아픈 느낌)"이라는 **다른 축**을 측정하기 때문
- 따라서 주관 지표의 종류에 따라 재는 것이 다르며, 이를 별도 축으로
  다루어야 한다는 결론이 도출된다

## 2.2 2×2 상태 분류 체계

두 독립 축을 교차하면 네 가지 상태가 정의된다.

| | **객관 양호** (PVT 정상) | **객관 위험** (PVT 저하) |
|---|---|---|
| **주관: 안 피곤** | ① 정상 | ② 숨은 위험 ⚠ |
| **주관: 피곤** | ③ 주관적 피로 | ④ 피로 확실 |

- **① 정상:** 객관·주관 모두 양호 → 정상 근무 가능
- **② 숨은 위험:** 객관적으로 위험하나 본인은 인지 못함 → **최우선 경고 대상**
- **③ 주관적 피로:** 본인은 피곤하나 객관 지표는 양호 → 휴식 권고, 심리적 지원
- **④ 피로 확실:** 객관·주관 모두 위험 → 즉시 작업 중단·조치

이 체계의 핵심 가치는 **② 숨은 위험을 별도로 식별**한다는 점이다.
기존 방식(주관 단독 또는 객관 단독)으로는 이 그룹을 놓치거나 정상으로 오인한다.

## 2.3 개인 baseline 개념

객관 축 판정 시, 집단 평균이 아니라 **개인의 평소 상태(baseline) 대비**로
판정해야 한다. 원래 반응이 느린 사람과 오늘 나빠진 사람을 구분하기 위함이다.
(상세 근거는 4장 및 향후 과제에서 기술)

## 2.4 투트랙(Two-Track) 서비스 모델

사용 주체와 상황에 따라 두 개의 모델을 제공한다.

| 구분 | Model 1 (근로자 자가진단) | Model 2 (안전관리자용) |
|---|---|---|
| 대상 | 근로자 본인 | 안전관리자 |
| 입력 | 개인특성 + 당일 간편 설문 | + PVT 3분 측정 |
| 출력 | 주관 피로 수준 안내 | 2×2 상태 정밀 판별 |
| 목적 | 간편 일상 체크 | 숨은 위험 식별·관리 |

Model 1은 PVT 없이 간편하게 자가 점검하는 용도이며,
Model 2는 PVT를 포함해 객관·주관을 모두 판별하는 정밀 버전이다.

# 3. 아키텍처 (Architecture)

## 3.1 솔루션 아키텍처 (서비스 흐름)

### 3.1.1 전체 파이프라인

```
[입력]  개인특성 등록(가입 시) + 당일 설문 + PVT 측정(선택/정밀)
   ↓
[전처리]  결측·이상치 처리, 표준화, (운영 시) 개인 baseline z-score 변환
   ↓
[판별]  판별기1(객관 각성) + 판별기2(주관 피로) 각각 산출
   ↓
[교차]  2×2 상태 분류 (정상/숨은위험/주관적피로/피로확실)
   ↓
[출력]  근로자 안내 + 안전관리자 대시보드 경고
```

### 3.1.2 대상별 상세

| 단계 | 근로자용 (Model 1) | 안전관리자용 (Model 2) |
|---|---|---|
| 입력 데이터 | 기본정보(연령·성별), 당일 설문(피로·졸림) | + PVT 3분 반응속도 측정 |
| 검사 방식 | 설문 기반 간편 예측 | 설문 + PVT 통합 정밀 판별 |
| AI 예측 | 주관 피로 수준(Model 1) | 2×2 상태 분류(Model 2) |
| 결과 리포트 | 피로 수준 + 관리 가이드 | 상태 분류 + 위험도 시각화 + 숨은위험 경고 |
| 특화 기능 | 사용자 친화 간편 체크 | 팀 대시보드, 우선순위 알림 |

## 3.2 기술 아키텍처

| 계층 | 기술 요소 | 용도 |
|---|---|---|
| 입력처리 | Streamlit Form | PVT 측정 UI, 설문 입력 폼 |
| 데이터 전처리 | Pandas, NumPy | 정제, 5분→3분 절삭, 결측 처리 |
| | StandardScaler | 변수 표준화 |
| | VIF 분석 (statsmodels) | 다중공선성 제거 |
| 모델링 | scikit-learn | Logistic, RandomForest, GradientBoosting |
| | XGBoost, LightGBM | 부스팅 계열 비교 |
| | 5-fold StratifiedKFold | 교차검증 |
| 개인화 | z-score (baseline) | 개인 대비 판정 (운영 시) |
| 결과처리 | 2×2 분류 로직 | 4상태 판별 |
| 해석·시각화 | SHAP | 변수 영향도 분석 |
| | Matplotlib | 위험도 시각화 |
| 배포 | Streamlit | 웹 대시보드 |

## 3.3 데이터 흐름의 근거

- **PVT 3분 채택:** 측정시간 검증(4.5절)에서 3분이 5분과 동등한 성능을 보여
  사용자 부담을 최소화하면서 정확도를 유지하는 최적점으로 확인됨
- **투트랙 분리:** 근로자는 매일 간편하게(설문), 정밀 판별이 필요할 때
  안전관리자가 PVT를 포함해 판별하는 2단계 구조

# 4. 모델 설계 및 개발 (Model Design & Development)

## 4.1 데이터 분석 플로우

```
STEP 1  데이터 정제 (PVT 원시 → 5분 통일, 파생변수)
STEP 2  탐색적 분석 (EDA) — 수면박탈 효과, 상관구조
STEP 3  변수 선택 — VIF(PVT) + 통계검정(설문)
STEP 4  모델링 — 알고리즘 비교, 하이퍼파라미터 튜닝
STEP 5  검증 — 5-fold 교차검증, 측정시간 최적화
STEP 6  구조 검증 — 2×2 분류, 숨은 위험 확인
```

## 4.2 데이터 전처리 (STEP 1)

### 4.2.1 원시 데이터 구조
PVT 원시 파일은 자극 제시 시각(onset)과 반응 시간(response_time)만 포함.
참가자당 2세션(정상수면 / 부분 수면박탈), 총 169세션.

### 4.2.2 정제 처리
| 문제 | 처리 방법 |
|---|---|
| onset 값에 `\r`(캐리지리턴) 혼입 → 데이터 밀림 | lineterminator 명시로 정확히 파싱 |
| 세션 길이 불일치 (41~98 trial) | 처음 5분(300초)으로 통일 절삭 |
| 반응 유형 구분 | <100ms=false start, 100~500ms=유효, ≥500ms=lapse |
| 세션별 trial 수 차이 | lapse는 개수 대신 비율로 정규화 |

## 4.3 변수 설계 (STEP 3)

### 4.3.1 PVT 파생변수 선택 (13개 → 5개)

원시 데이터에서 13개 지표를 계산했으나, 이들은 같은 반응시간을
여러 방식으로 요약한 것이라 서로 강하게 겹친다(다중공선성).

**공선성 확인 (VIF, 선정 전):**
- `lapse_rate` 356.6, `mean_rt` 179.9, `sd_rt` 170.8 등 대부분 10 초과
- `n_trial`, `n_valid`, `false_starts`, `lapses`는 정의상 완전 종속(VIF ∞)

**상관구조로 5개 축 도출 (|r|≥0.8 기준):**
| 축 | 포함 변수 | 대표 선정 |
|---|---|---|
| 반응속도 | mean_rt, median_rt, mean_speed, slowest10_rt 등 | `mean_rt` |
| 변동성 | sd_rt, cv_rt | `sd_rt` |
| 주의실패 | lapses, lapse_rate | `lapses` |
| 반응량 | n_valid, n_trial | `n_valid` |
| 성급함 | false_starts | `false_starts` |

**선정 후 VIF:** 전부 1.1~1.4 (공선성 완전 해결)

### 4.3.2 설문 변수 선택 (106개 → 9개)

participants.tsv의 106개 컬럼을 4단계로 필터링.

| 필터 | 내용 |
|---|---|
| 1. 부적합 제외 | 식별자·실험통제·텍스트형·타깃계열 (~25개) |
| 2. 결측 과다 | MMSE(결측 55%) 등 제외 |
| 3. 하위척도 정리 | KSQ 18개 등 대표 지표만 |
| 4. 통계검정 | p<0.05 또는 |r|≥0.3 |

**채택된 설문 변수 (타깃 SicknessQ와의 상관):**
| 변수 | 설명 | r | p |
|---|---|---|---|
| `STAI_T` | 특성불안 | +0.475 | <0.001 |
| `PSS14` | 스트레스 | +0.375 | <0.001 |
| `ISI` | 불면 | +0.359 | <0.001 |
| `BFI_Neuroticism` | 신경증 | +0.300 | <0.001 |
| `HADS_Anxiety` | 임상불안 | +0.299 | <0.001 |
| `TAS20_Total` | 감정표현불능 | +0.284 | <0.001 |
| `BIS11_Attentional` | 주의충동 | +0.280 | 0.001 |
| `BFI_Conscientiousness` | 성실성 | −0.157 | 0.042 |
| `AgeGroup` | 연령군 | χ²=8.95 | 0.003 |

**실용성 제외:** MMSE(결측), EducationLevel(민감정보), ESS(타깃과 무관 r=0.004)

### 4.3.3 타깃 변수 선정 근거

## 4.4 모델링 및 성능 개선 (STEP 4)

### 4.4.1 알고리즘 비교 (5-fold CV, AUC)

| 모델 | 계열 | AUC | 비고 |
|---|---|---|---|
| Logistic Regression | 선형 | 0.746 | 기준선(baseline) |
| AdaBoost | 부스팅 | 0.777 | |
| RandomForest | 배깅 | 0.792 | |
| XGBoost | 부스팅 | 0.770 | |
| XGBoost (튜닝) | 부스팅 | 0.797 | |
| LightGBM | 부스팅 | 0.772 | |
| **GradientBoosting** | 부스팅 | **0.806** | **최종 선정** |

### 4.4.2 성능 개선 과정
1. Logistic Regression으로 기준선 확보: 0.746
2. 트리 계열 전환 → 비선형·상호작용 포착: 0.79대
3. 하이퍼파라미터 튜닝(GridSearchCV): GradientBoosting 0.806

### 4.4.3 중요한 방법론적 발견
- **XGBoost/LightGBM(0.77)보다 GradientBoosting(0.81)이 우수**
- 이유: 표본이 169세션으로 작아, 대용량에 강한 XGBoost류보다
  기본 GradientBoosting의 정규화가 더 효과적
- 시사점: 최신 기법이 항상 최선은 아니며 데이터 규모에 맞는 선택이 중요

### 4.4.4 모델 선정 기준
- **1차 기준: Recall(재현율)** — 위험 근로자를 놓치지 않는 것이 최우선
  (산업안전에서 False Negative는 사고로 직결되는 치명적 오류)
- **2차 기준: 전반 성능(AUC)** — Recall 동률 시 적용

### 4.4.5 변수 중요도 (최종 모델)
| 순위 | 변수 | 중요도 |
|---|---|---|
| 1 | 특성불안(STAI_T) | 0.30 |

| 3 | PVT 변동성(sd_rt) | 0.14 |

## 4.5 측정시간 최적화 (STEP 5 — 서비스 설계 핵심)

사용자 부담을 최소화할 최적 측정시간을 결정하기 위해
PVT를 60/90/180/300초로 각각 재계산하여 성능 비교.

| 측정시간 | trial 평균 | PVT+개인특성 AUC |
|---|---|---|
| 60초 | 10.1 | 0.721 (개인특성만보다 낮음) |
| 90초 | 14.8 | 0.761 |
| **180초 (3분)** | 29.4 | **0.799** |
| 300초 (5분) | 46.6 | 0.806 |

**결론:**
- 60초는 trial이 부족해 오히려 노이즈 (개인특성만 0.738보다 낮음)
- 3분이 5분과 거의 동일(차이 0.007)하면서 부담 절감 → **3분 최적**
- 90초까지 타협 가능

## 4.6 검증 방식 (train/test)

- **5-fold 층화 교차검증** (StratifiedKFold, shuffle, random_state=42)
- 매 fold: train 약 80%(135~136개) / test 약 20%(33~34개), 5회 반복
- 모든 세션이 정확히 1번씩 test에 사용, 결과는 5개 평균
- Stratified로 피로 높음/낮음 비율을 전체와 동일하게 유지
- 소표본에서 고정 분할보다 안정적 (참고자료 팀도 K-Fold 사용)

# 6. 핵심 검증 결과: 2×2 구조와 숨은 위험

## 6.1 숨은 위험은 실재하는가

두 축(객관 PVT, 주관 SicknessQ)을 각각 중앙값 기준으로 나눠 교차한 결과:

| 상태 | 세션 수 | 비율 |
|---|---|---|
| ① 정상 | 36 | 21.3% |
| ② 숨은 위험 | 47 | 27.8% |
| ③ 주관적 피로 | 48 | 28.4% |
| ④ 피로 확실 | 38 | 22.5% |

→ **숨은 위험이 27.8% 실재.** 전체의 약 1/4이 "본인은 인지 못하나
객관적으로 각성이 저하된" 상태.

## 6.2 세 가지 통계 검증 (모두 통과)

### 검증 1 — 두 축의 독립성
- 카이제곱 χ²=2.14, p=0.143 → **독립**
- 두 축이 다른 정보를 담으므로 숨은 위험이 자연 발생

### 검증 2 — 숨은 위험 그룹은 실제로 위험한가
| 지표 | 숨은위험 | 정상 | p |
|---|---|---|---|
| 평균반응시간 | 271.7ms | 249.3ms | <0.001 |
| 주의실패 | 1.0 | 0.2 | 0.001 |
| 변동성 | 53.1 | 39.4 | <0.001 |
→ 정상보다 객관 각성이 확실히 저하

### 검증 3 — 숨은 위험 vs 피로 확실 (핵심)
| 지표 | 숨은위험 | 피로확실 | p |
|---|---|---|---|
| 객관: 반응시간 | 271.7 | 279.4 | 0.172 (차이 없음) |
| 주관: 피로점수 | 14.5 | 22.1 | <0.001 (완전 다름) |
→ **두 그룹 모두 객관적으로 위험하나, 숨은 위험만 그것을 인지 못함.**
   이 "위험 인지 실패"가 산업안전의 핵심 리스크.

## 7.2 원래 느린 사람 문제 (중요한 한계)

집단 중앙값 기준으로 판정하면, 원래 반응이 느린 사람은 컨디션과 무관하게
항상 "객관 위험"으로 분류되는 착시가 발생한다.

**데이터로 확인:**
- 숨은위험 47세션 중 자기 평소보다 실제 나빠진 "진짜 위험"은 10개
- 나머지 37개는 원래 느리거나 안 나빠진 경우
- 정상수면 상태인데도 객관위험 판정된 세션 42개 = 원래 느린 사람

**해결 방향:** 개인 baseline 기반 z-score
```
z = (오늘 PVT − 그 사람 평소 평균) / 그 사람 평소 표준편차
```
→ 원래 느린 사람도 "자기 평소만큼"이면 정상, "평소보다 나빠지면" 위험

## 7.3 향후 서비스 운영으로 검증할 것 (제안)

| 제안 | 왜 지금 증명 불가한가 |
|---|---|
| 개인 baseline 축적 → z-score 개인화 | 1인 2세션뿐, 반복측정 부족 |
| baseline 확립 후 정확도 향상 | 장기 반복측정 데이터 필요 |
| 사용할수록 개인화되어 성능 개선 | 실제 운영 데이터로만 확인 가능 |

이는 근거 없는 기대가 아니라, 7.2절(집단 기준의 한계)에서 논리적으로
도출된 **근거 있는 설계 제안**이다. 효과의 정량 검증은 실제 서비스
운영 데이터를 통한 향후 과제로 남긴다.

## 7.4 데이터의 본질적 한계

- ds000201은 원래 감정 처리 연구용이며 PVT는 부수 측정 → PVT 신호 약함
- 부분 수면박탈(3시간) + 저녁 측정(각성 회복 시간대)이라 객관 효과 제한적
- 표본 규모(85명, 169세션)가 크지 않음
- 주관적 피로는 본질적으로 노이즈가 많음

이러한 한계에도 불구하고, 본 연구는 "피로의 이중 구조"와 "숨은 위험"이라는
개념을 데이터로 입증하고, 산업안전에 적용 가능한 서비스 구조를 제시했다는
점에서 의의가 있다.
