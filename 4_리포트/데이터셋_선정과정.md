# 데이터셋 선정 과정 및 근거

> 프로젝트 목적: PVT + 주관 피로/졸림 + 개인특성으로 피로·각성 상태를 판별하는
> 산업안전 서비스. 이를 위해 "PVT 원시데이터 + 주관 지표"를 갖춘 데이터가 필요.

|---|---|---|---|---|---|
| **ds000201** (스톡홀름 Sleepy Brain) | 85명/169세션 | 원시(onset) | SicknessQ | ✅ **주 데이터** | PVT 원시 완비, 측정시간 분석 가능, 결측 적음 |

## 2. 각 데이터셋 상세 검토

### 2-1. ds000201 — 주 데이터로 채택
- **장점:** PVT 원시 이벤트(onset, response_time) 완비 → 파생변수 자유 생성,
  측정시간별 분석(60/90/180/300초) 가능, 85명으로 이 분야 적정 규모, 결측 적음
- **한계:** 부분 수면박탈(3시간)+저녁 측정이라 PVT 신호 약함,
  주관지표가 SicknessQ(아픈느낌)로 '졸림·각성'과는 무관
- **활용:** 전처리·EDA·변수선택·모델링의 주 데이터

### 2-2. DROZY — 보조 데이터로 채택
- **구조:** 14명 × PVT 3세션(오전 10시/새벽 3시/낮 12시), 각 세션당 KSS 1개
- **장점:** KSS(진짜 주관 졸림)와 PVT가 세션별로 짝지어짐,
  급성 수면박탈(밤샘)이라 각성 변화 뚜렷(KSS 3→6→7), 결측 거의 없음
- **한계:** 42세션으로 규모 작음 → 머신러닝 주 데이터로는 부적합
- **활용:** "주관 졸림(KSS) ↔ 객관 각성(PVT)" 관계 검증 (2×2 구조 근거)

### 2-3. ds004902 — 제외
- KSS·SSS 주관 졸림이 있어 매력적이었으나,
- **KSS와 PVT를 둘 다 가진 참가자가 71명 중 단 2명** (결측이 엇갈림)
- 요약 테이블 기준 KSS 33명·PVT 31명이나 거의 겹치지 않음
- KSS는 설문이라 원시 복구 불가 → 핵심 목적(KSS-PVT 관계) 분석 불가

### 2-4. EPOC (ds007605) — 제외
- 코로나19 후유증 환자 대상, FACIT는 '지난 7일' 만성 피로(상태 아닌 특성)
- 우리의 '그 순간 상태' 예측 목적과 시간 축이 다름

### 2-5. Kaggle CSV — 제외
- 결측 0, 모든 값이 고유 등 합성 데이터 정황
- PVT가 참가자당 평균값 1개(원시 없음) → 파생변수 생성 불가
- 개발 초기 파이프라인 테스트용으로만 가치

### 2-6. UL-DD — 제외
- KSS를 4분마다 측정해 매력적이나 **PVT 자체가 없음**
- 운전 시뮬레이터 + 생체신호(심박·피부전도) + 얼굴영상 중심
- 우리 PVT 기반 설계와 근본적으로 다름

## 3. 최종 선정: ds000201(주) + DROZY(보조)

**역할 분담**
- **ds000201:** 전처리 파이프라인, EDA, 변수선택, VIF, 측정시간 최적화,
  머신러닝 모델링 → 프로젝트의 정량적 본체
- **DROZY:** KSS-PVT 관계 검증 → "주관과 객관을 각각 봐야 한다"는
  2×2 구조의 이론적 근거 제공 (SicknessQ가 각성과 무관했던 이유 설명)

**선정 원칙**
1. PVT 원시데이터 필수 (파생변수·측정시간 분석을 위해)
2. 주관 지표가 '상태'여야 함 (특성 지표 제외)
3. 결측이 분석 가능한 수준
4. "완벽한 단일 데이터"는 존재하지 않으므로, 강점이 다른 둘을 역할 분담

- 6종 데이터를 체계적으로 검토한 과정 자체가 데이터 리터러시의 근거
- "왜 이 데이터인가"를 명확한 기준(원시성·상태성·결측)으로 방어 가능
- ds004902·EPOC 제외 사유는 "데이터를 꼼꼼히 검증했다"는 신뢰로 작용

## 5. DROZY 검증 결과 (보조 데이터 분석 완료)

14명 × PVT 3세션(오전/새벽/낮) = 36세션 분석.

### 5-1. 수면박탈 진행에 따른 변화

| 테스트 | KSS(주관) | PVT lapse(객관) | 평균RT | 변동성 |
|---|---|---|---|---|
| 오전 10시(각성) | 3.3 | 4.3 | 358ms | 63 |
| 새벽 3시(밤샘) | 5.6 | 15.0 | 423ms | 127 |
| 낮 12시(회복중) | 7.3 | 23.6 | 464ms | 163 |

→ 주관·객관 지표가 나란히 악화

### 5-2. KSS ↔ PVT 상관 (핵심 근거)

| PVT 지표 | KSS와의 상관 r | p |
|---|---|---|
| 주의실패(lapse) | +0.634 | <0.001 |
| lapse 비율 | +0.629 | <0.001 |
| 느린10% | +0.619 | <0.001 |
| 평균반응시간 | +0.616 | <0.001 |
| 반응변동성 | +0.580 | <0.001 |

### 5-3. 2×2 구조의 근거 확립

- **DROZY:** 졸림 척도(KSS) ↔ 객관 각성(PVT) = 강한 상관 (r≈0.63)
- **ds000201:** 신체 척도(SicknessQ) ↔ 객관 각성(PVT) = 무관 (r≈0)
- **결론:** 주관-객관 관계는 "어떤 주관 척도를 쓰느냐"에 좌우됨
  - 졸림 척도는 각성을 반영하나, 신체 컨디션 척도는 별개 축
  - → 산업안전 서비스는 두 축을 각각 판별해야 함 (2×2 구조)
  - → 특히 "주관은 안 졸린다는데 객관은 위험한" 숨은 위험 탐지가 핵심

## 6. DROZY의 역할 명확화 (자주 나오는 질문 대비)

### Q: DROZY는 모델 학습에 안 쓰는데 왜 필요한가?

DROZY는 **모델 학습용이 아니라, 2×2 구조의 정당성을 방어하는 검증 데이터**다.

**반박 시나리오:** "ds000201에서 SicknessQ와 PVT가 무관(r≈0)한 건,
SicknessQ가 엉터리 측정이라서 아닌가? 그럼 '숨은 위험'도 허구 아닌가?"

**DROZY로 방어:** 주관 지표가 엉터리라서 무관한 게 아니다.
- DROZY에서 **졸림 척도(KSS) ↔ PVT = r=0.63 (강한 상관)**
- 즉 주관을 제대로 재면(졸림) 객관과 잘 연관됨
- SicknessQ가 PVT와 무관한 이유는 "졸림"이 아니라 "신체 컨디션"이라는
  **다른 축**을 재기 때문
- → 두 축을 나눠 보는 2×2 구조가 타당하다는 근거

### 정리: 두 데이터의 역할

| 데이터 | 역할 | 용도 |
|---|---|---|
| ds000201 | 주 데이터 | 정제·EDA·변수선택·모델학습·2×2 검증 |
| DROZY | 대조군 | "주관-객관은 원래 연관 가능"을 입증 → 2×2 정당화 |

DROZY 없이 ds000201만으로는 "SicknessQ의 독립성이 측정오류인지
진짜 다른 축인지" 구분 불가 → DROZY가 이 논리를 완성함.
