## A. PVT 최소 측정시간 검증 (기획 결정 ①)

**목적:** 사용자 부담 최소화를 위한 최적 측정시간 결정

### A-1. 측정시간별 데이터 특성

| 측정시간 | trial 평균 | lapse 평균 |
|---|---|---|
| 60초 | 10.1회 | 0.16 |
| 90초 | 14.8회 | 0.20 |
| 3분 | 29.4회 | 0.36 |
| 5분 | 46.6회 | 0.57 |

### A-2. 측정시간별 피로도 예측 성능 (AUC, GradientBoosting)

| 측정시간 | PVT만 | PVT+개인특성 | 개인특성만(참조) |
|---|---|---|---|
| 60초 | 0.599 | 0.721 | 0.738 |
| 90초 | 0.616 | **0.761** | 0.738 |
| 3분 | 0.557 | **0.799** | 0.738 |
| 5분 | 0.662 | **0.806** | 0.738 |

### A-3. 핵심 발견 및 기획 시사점

- **60초는 부적합:** PVT trial이 10개뿐이라 지표가 불안정.
  PVT를 더해도 개인특성만(0.738)보다 낮아짐(0.721) = 노이즈로 작용
- **90초부터 PVT가 기여 시작:** 0.761 > 0.738
- **3분이 균형점:** 0.799로 5분(0.806)과 거의 동일(차이 0.007)하면서 부담 절감
- **권장:** 기본 3분, 부담 최소화가 우선이면 90초까지 타협 가능

## B. 설문변수 상세 및 서비스 입력 설계 (기획 결정 ②)

### B-1. 초기 정보 (회원가입 시 1회 입력) — 특성 변수

| 변수 | 측정 내용 | 실측범위 | 서비스 활용 코멘트 |
|---|---|---|---|
| 특성불안(STAI) | 평소 불안 성향 | 20~51 | **최강 예측자.** 변별력 큼(SD 6.9). 필수 |
| 지각스트레스(PSS) | 최근 스트레스 | 1~35 | 변별력 큼(SD 6.9). 권장 |
| 주간졸림(ESS) | 평소 조는 정도 | 0~21 | 변별력 있음(SD 4.2). 권장 |
| 신경증(BFI) | 정서 불안정성 | 8~27 | 예측 기여. 성격검사 형태 |
| 불면(ISI) | 불면 증상 | 7~14 | **주의: 범위 좁음.** 원연구가 불면자 제외해서 변별력 약할 수 있음 |
| 우울(HADS) | 우울 증상 | 0~7 | 범위 좁음(SD 1.2). 보조용 |

### B-2. 일일 정보 (매일 측정 시 입력) — 상태 변수

| 변수 | 서비스 활용 |
|---|---|
| 전날 수면시간 | baseline 대비 당일 변화 핵심 입력 |
| 당일 졸림(주관) | KSS 유형. 당일 컨디션 |
| (PVT 결과) | 매 측정 시 자동 산출 |

**주의:** 원 데이터에는 전날 수면시간(PSG)은 있으나 당일 주관졸림(KSS)은 없음.

## C. 변수선택 과정 및 VIF (다중공선성) 처리

### C-1. 문제: PVT 파생변수 간 심각한 공선성

초기 PVT 파생변수 13개의 VIF (10 초과 = 심각):

| 변수 | VIF | 판정 |
|---|---|---|
| n_trial, n_valid, false_starts, lapses | ∞ (무한대) | 정의상 상호종속 |
| lapse_rate | 356.6 | 제거 |
| mean_rt | 179.9 | 제거 |
| sd_rt | 170.8 | 제거 |
| mean_speed | 46.0 | 제거 (mean_rt의 역수) |
| slowest10_rt | 20.9 | 제거 |

원인: 변수들이 서로의 계산으로 생성됨
(예: n_trial = n_valid + false_starts + lapses, mean_speed = 1/mean_rt)

### C-2. 해결: 축별 대표변수 선정

상관구조 분석으로 4개 독립 축 확인 → 각 축 대표 1개 선정:

| 축 | 대표 변수 | 선정 후 VIF |
|---|---|---|
| 반응속도 | mean_rt | 1.41 |
| 변동성 | sd_rt | 1.33 |
| 주의실패 | lapses | 1.28 |
| 반응량 | n_valid | 1.23 |
| 성급함 | false_starts | 1.15 |

→ 전부 VIF < 2 (공선성 완전 해결). 이 5개를 PVT 특징으로 확정.

### D-2. 성능개선 과정

1. 로지스틱 회귀 baseline: 0.746
2. 트리 계열 전환 (비선형·상호작용 포착): 0.79대
3. 부스팅 계열 폭넓게 비교: AdaBoost, XGBoost, LightGBM, GradientBoosting

### D-3. 중요한 방법론적 발견

- **XGBoost/LightGBM이 GradientBoosting보다 낮음** (0.77 vs 0.81)
- 이유: 표본이 작음(169행). XGBoost류는 대용량에서 강점,
  소규모에서는 기본 GradientBoosting의 정규화가 더 효과적
- **시사점:** 최신 기법이 항상 최선은 아니며, 데이터 규모에 맞는 선택이 중요

1. **측정시간:** 3분 권장 (성능·부담 균형), 90초까지 타협 가능, 60초는 부적합
2. **입력설계:** 초기 1회(특성불안·스트레스·졸림) + 일일(수면시간·졸림·PVT)

## F. 모델 검증 방식 (train/test 분할)

### F-1. 검증 방법: 5-fold 층화 교차검증

**train/test를 고정 비율로 1회 나누지 않고, 5-fold 교차검증(Stratified K-Fold) 사용.**

- 설정: `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`
- 전체 169세션 (피로낮음 83 / 피로높음 86)

### F-2. 각 Fold 구성

| Fold | Train | Test | Test 내 피로높음 |
|---|---|---|---|
| 1 | 135 | 34 | 18/34 |
| 2 | 135 | 34 | 17/34 |
| 3 | 135 | 34 | 17/34 |
| 4 | 135 | 34 | 17/34 |
| 5 | 136 | 33 | 17/33 |

- **매 fold: train 약 80% (135~136개) / test 약 20% (33~34개)**
- 5회 반복하여 모든 세션이 정확히 1번씩 test에 사용됨
- 최종 성능(AUC)은 5개 fold 결과의 평균

### F-3. 왜 고정 holdout 대신 교차검증인가

- 표본이 169세션으로 작음 → 1회 분할 시 test(34개)가 우연에 크게 흔들림
- 교차검증은 전체 데이터를 검증에 활용 → 작은 표본에서 더 안정적·신뢰성 높음
- **Stratified** 적용: 각 fold의 피로높음/낮음 비율을 전체(약 51:49)와 동일하게
  유지하여 클래스 불균형으로 인한 편향 방지
- 참고자료(CAD팀)도 K-Fold 기반 최종 모델 선정 → 동일 방법론

### F-4. 재현성
- `random_state=42` 고정 → 동일한 fold 분할 재현 가능
